# Pull Request Reviews

> **When to Apply:** When asked to review a pull request.

## Review Process

1. **Read the PR description** to understand the intended changes

2. **Analyze the code changes**

   - Consider edge cases relevant to the changes
   - Evaluate if the proposed changes fully address the PR intentions
   - Look for potential bugs, performance issues, or security concerns

3. **Leave thoughtful feedback**

   - If unsure about certain changes, call attention to it in a comment anyway
   - Reference specific code in your comments
   - Ask follow-up questions if needed

4. **Submit the review** using the GitHub MCP
   - Use appropriate review status (approve, request changes, or comment)
