# Code Changes

> **When to Apply:** When the task requires implementing or modifying code.

## Workflow

1. **Clone/checkout** the repository if you haven't already

   - If working on an existing PR, checkout that PR's branch directly
   - If working on an issue or new feature, create a new branch with a descriptive name

2. **Implement** the requested changes

3. **Verify** your changes (see verification instructions)

4. **Commit** with a clear message following conventional commit format

5. **Check** for style or linting tools like prettier and fix any errors

6. **Push** your changes

7. **Create or update PR**
   - If working on an existing PR branch, the PR will automatically update
   - If you created a new branch, create a pull request using the GitHub MCP
